# Game Activity Config Validator - 4 Layer Configs

这组配置文件对应“基于 checklist 的策划配置表业务级联表校验工具”。

## 四层配置

1. `01_schema.yaml`：表映射层，维护 10 张策划配置表的主键、字段和外键关系。
2. `02_rule_library.yaml`：规则模板层，按 4 类规则模式维护可复用 Checker 模板。
3. `03_presets.yaml`：业务校验包层，把通用规则实例化为活动综合检查、红点专项、签到专项、兑换奖池专项。
4. `04_checks_summer_night.yaml`：QA 业务脚本层，普通 QA 只需要指定活动 ID 和 preset。

## 工具边界

本工具不替代导表工具，也不替代游戏运行时测试。

- 不做：字段类型、基础非空、基础枚举、资源加载表现。
- 不能做：UI 实际渲染、动画播放、玩家点击后的真实状态变化。
- 专注做：活动时间、红点、签到、货币、兑换、奖池、奖励物品等配置之间的业务级一致性校验。

## 对应 checklist 的核心场景

- 日期 CL：活动总表、签到表、任务表、兑换表、奖池表的时间窗口一致性。
- 红点 CL：活动结束后红点残留、一级/二级红点父子关系、红点开放等级。
- 逻辑 CL：签到天数、同账号/同角色领取维度、中途未上线补领策略。
- 夜之玉：周上限 120、周一刷新、任务产出货币与兑换消耗货币一致。
- 奖励兑换：展示奖励与实际奖励一致、奖池下一阶段/预览链路、奖池刷新红点。
